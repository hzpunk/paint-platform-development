# Техническое задание: Административная панель платформы для продажи красок

> **Назначение документа:** ТЗ для разработчиков, дизайнеров и нейросети, реализующей административную панель (backoffice) платформы.
> **Связанный документ:** ТЗ на клиентскую часть платформы (`tz_paint_platform.md`).
> **Правовая база:** Законодательство РФ — ФЗ «О защите прав потребителей», ФЗ «О персональных данных» (152-ФЗ), правила дистанционной торговли (ПП РФ № 612), требования Роспотребнадзора к продаже ЛКМ, 54-ФЗ (в части документооборота по наложенным платежам).

---

## 1. Концепция и позиционирование

**Продукт:** Внутренняя административная панель (backoffice) для управления интернет-магазином ЛКМ — каталогом, заказами, клиентами, программой лояльности, оптовыми продажами и контентом.

**Пользователи панели:**
- Владелец/руководитель — полный доступ, аналитика
- Менеджер по заказам — обработка заказов, коммуникация с клиентами
- Контент-менеджер — каталог, блог, цены
- Менеджер по колеровке — заказы на колеровку
- Оператор оптовых продаж — заявки B2B
- Бухгалтер — финансовые отчёты, документы
- Модератор — отзывы, UGC-контент

**Ключевые ценности:** Скорость обработки заказов, полнота данных для принятия решений, минимизация ручных операций за счёт автоматизации.

**Задачи панели:**
- Управление каталогом товаров и остатками
- Обработка заказов от создания до доставки
- Управление программой лояльности и рефералкой
- Работа с оптовыми клиентами и B2B-документами
- Модерация отзывов и контента
- Аналитика продаж, клиентов, эффективности маркетинга
- Управление правами доступа сотрудников

---

## 2. Стиль и дизайн-система

### 2.1 Общая стилистика

**Направление:** Строгий рабочий интерфейс данных (data-dense dashboard). В отличие от клиентской части — минимум декоративности, максимум плотности полезной информации и скорости навигации.

**Настроение:** Функциональность, предсказуемость, контроль.
**Референсы:** Retool, Linear (структура навигации), Metabase (аналитика/таблицы), внутренние панели Ozon/Wildberries Seller (плотность данных).

Дизайн-система наследует брендовую палитру и типографику клиентской части (раздел 2.2–2.3 основного ТЗ), но адаптирована под рабочий интерфейс с поддержкой светлой и тёмной тем.

### 2.4 UI-компоненты

- **Скругления:** 6px — карточки и таблицы, 6px — кнопки, 4px — инпуты, 10px — модалки
- **Тени:** минимальные, `0 1px 3px rgba(0,0,0,0.06)` — только для выпадающих меню и модалок
- **Плотность:** компактные строки таблиц (36–40px высота), возможность переключения «компактный / комфортный» режим
- **Иконки:** Lucide Icons
- **Сетка:** сайдбар 260px (сворачиваемый до 64px) + основная область, max-width контента не ограничен (data-first)
- **Таблицы:** закреплённая шапка, сортировка по колонкам, выбор строк чекбоксами, массовые действия, настраиваемые колонки
- **Уведомления:** toast (действия), колокольчик с бейджем (системные события)

## 3. Страницы административной панели

---

### 3.1 Авторизация и структура (`/admin/login`)

- Вход по email/логину + паролю, обязательная 2FA (SMS или TOTP-приложение) — из-за работы с ПД (152-ФЗ)
- Восстановление пароля
- Журнал входов (IP, устройство, время) — доступен администратору

**Общий каркас панели:**
- Сайдбар слева: разделы меню (см. ниже), сворачиваемый
- Верхняя панель: поиск по всей системе, уведомления, переключение темы, профиль/выход
- Хлебные крошки под верхней панелью

---

### 3.2 Дашборд (`/admin`)

**Цель:** Общая картина состояния бизнеса за выбранный период на одном экране.

**Блоки:**
1. **KPI-карточки** (за период, с сравнением к прошлому периоду): выручка, кол-во заказов, средний чек, конверсия, новые клиенты
2. **График продаж** (по дням/неделям/месяцам, с фильтром по категориям)
3. **Заказы, требующие внимания** — просрочка обработки, отменённые, возвраты, недозвон
4. **Топ товаров** (по продажам и по остаткам на исходе)
5. **Товары с низким остатком** — список с быстрым переходом к пополнению
6. **Новые заявки оптовиков** и **новые заказы на колеровку** — счётчики + быстрый переход
7. **Свежие отзывы, ожидающие модерации**
8. **Виджет активности сотрудников** (кто что обработал за смену) — для руководителя

---

### 3.3 Заказы (`/admin/orders`)

**Список заказов:**
- Таблица: № заказа, дата, клиент, сумма, статус, способ доставки, способ оплаты, менеджер
- Фильтры: статус, период, способ доставки (СДЭК/Boxberry/Почта/Самовывоз), сумма, оптовый/розничный
- Массовые действия: смена статуса, экспорт в Excel, печать накладных

**Статусы заказа (воронка):**
`Новый → Подтверждён → Собирается → Передан в доставку → В пути → Доставлен → Оплачен` (при наложенном платеже) / `Отменён` / `Возврат`

**Карточка заказа (`/admin/orders/:id`):**
1. Шапка: № заказа, статус (с возможностью смены), дата, менеджер-ответственный
2. Данные клиента: имя, телефон, email, история заказов клиента, уровень лояльности
3. Состав заказа: товары, объём, цвет, кол-во, цена, наличие на складе
4. Доставка: адрес/ПВЗ, способ, стоимость, номер трек-кода (интеграция с СДЭК/Boxberry/Почта API), статус в реальном времени
5. Оплата: способ (наложенный платёж), сумма, статус оплаты
6. Применённые бонусы/промокоды
7. Внутренние комментарии менеджеров (не видны клиенту)
8. История изменений заказа (лог)
9. Действия: подтвердить, отменить (с указанием причины), оформить возврат, распечатать накладную/чек, отправить SMS/email клиенту

---

### 3.4 Каталог товаров (`/admin/catalog`)

**Список товаров:**
- Таблица: фото, название, артикул, категория, бренд, цена, остаток, статус (опубликован/скрыт), бейджи
- Фильтры: категория, бренд, наличие, статус публикации
- Массовые действия: изменение цены (в т.ч. процентом), изменение статуса, экспорт/импорт (CSV/Excel)

**Карточка товара — создание/редактирование (`/admin/catalog/:id`):**
1. Основное: название, артикул (SKU), категория, бренд, описание, статус публикации
2. Медиа: загрузка фото (drag&drop, множественная загрузка, сортировка), видео
3. Характеристики: тип ЛКМ, поверхность применения, состав, расход, время высыхания, укрывистость (кастомные атрибуты по категориям)
4. Расфасовки и цены: матрица «объём → цена → остаток → артикул расфасовки»
5. Цвета: доступные цвета/база для колеровки, привязка к RAL/NCS/Pantone
6. Документы: загрузка PDF (сертификат соответствия, паспорт безопасности) — обязательные поля, публикация недоступна без них (требование ГОСТ)
7. SEO: meta title, description, ЧПУ-адрес (slug)
8. Бейджи: Хит / Новинка / Акция (вручную или по правилу)
9. Похожие товары / «с этим покупают» — ручная или автоматическая привязка
10. История изменений цены и остатков

**Категории и атрибуты (`/admin/catalog/categories`):**
- Дерево категорий, drag&drop сортировка
- Управление наборами атрибутов/фильтров для каждой категории (то, что попадает в фильтры каталога на витрине)

**Управление остатками (`/admin/catalog/stock`):**
- Складской учёт по расфасовкам, приход/списание, история движения
- Пороги «мало на складе» с уведомлениями

---

### 3.5 Заказы на колеровку (`/admin/colormixing`)

- Список заявок: клиент, товар-база, целевой цвет/код, статус, менеджер
- Статусы: Новая → В расчёте → Подтверждена клиенту → Передана на производство → Готова → Включена в заказ
- Карточка заявки: параметры колеровки, прикреплённое фото цвета (если загружено клиентом), расчёт стоимости, комментарии
- Настройка тарифов на колеровку (по объёму базы)

---

### 3.6 Клиенты (`/admin/customers`)

**Список клиентов:**
- Таблица: имя, телефон, email, кол-во заказов, сумма покупок, уровень лояльности, дата регистрации
- Фильтры: уровень лояльности, розница/опт, активность (давно не покупал)
- Сегменты для рассылок (экспорт списка)

**Карточка клиента:**
1. Контактные данные, дата рождения (для ДР-скидки)
2. История заказов
3. Баланс и история бонусных баллов (начисления/списания вручную с указанием причины)
4. Уровень программы лояльности (с возможностью ручной корректировки)
5. Реферальная статистика: кто пригласил, кого пригласил сам, заработано баллов
6. Избранное и отзывы клиента
7. Внутренние заметки менеджера
8. Согласия на обработку ПД и рассылки (для 152-ФЗ — журнал согласий)

---

### 3.7 Программа лояльности и рефералка (`/admin/loyalty`)

1. **Настройка уровней:** пороги по сумме покупок, % начисления баллов, привилегии по уровню (Стандарт/Серебро/Золото/Платина)
2. **Правила начисления/списания:** курс балла к рублю, срок сгорания баллов, ограничения на списание (% от суммы заказа)
3. **Реферальная программа:** размер бонуса рефереру и приглашённому, особые условия для мастеров/бригад, лимиты
4. **Ручные операции:** начислить/списать баллы клиенту с указанием причины (для компенсаций)
5. **Отчёт:** общий объём начисленных/списанных/сгоревших баллов, стоимость программы для бизнеса

---

### 3.8 Оптовые клиенты (`/admin/wholesale`)

- Список заявок с сайта: компания, ИНН, контакты, статус (Новая/В обработке/Одобрена/Отклонена)
- Карточка заявки: данные компании, комментарии менеджера, история переписки
- После одобрения — привязка персональных условий: скидка от объёма, отсрочка платежа, персональный менеджер
- Прайс-листы: загрузка/генерация PDF для скачивания клиентом
- Закрывающие документы: генерация УПД, счёт-фактуры по оптовому заказу (интеграция с 1С при необходимости)

---

### 3.9 Отзывы и модерация (`/admin/reviews`)

- Список отзывов: товар, автор, оценка, текст, статус (На модерации/Опубликован/Отклонён), подтверждённая покупка
- Действия: одобрить, отклонить (с причиной), ответить от имени магазина
- Фильтр по товарам с низким рейтингом — для контроля качества

---

### 3.10 Блог и контент (`/admin/content`)

- Список статей: заголовок, рубрика, автор, статус (черновик/опубликовано), дата
- Редактор статей: WYSIWYG/Markdown, вставка карточек товаров внутрь статьи, SEO-поля, обложка
- Управление баннерами главной страницы (hero-баннер, акции)
- Управление статичными страницами (доставка, контакты, юридические — при необходимости редактирования без разработчика)

---

### 3.11 Промокоды и акции (`/admin/promotions`)

- Список промокодов: код, тип скидки (%, фикс. сумма, бесплатная доставка), условия, срок действия, лимит использований, статус
- Создание акции: применение к товарам/категориям, период действия, автоматическое проставление бейджа «Акция» на витрине
- Статистика использования промокодов

---

### 3.12 Аналитика и отчёты (`/admin/analytics`)

1. **Продажи:** динамика выручки, заказов, среднего чека по периодам, каналам, категориям
2. **Товары:** ABC-анализ, топ/аутсайдеры продаж, оборачиваемость склада
3. **Клиенты:** LTV, повторные покупки, эффективность программы лояльности, воронка регистрация → первая покупка
4. **Маркетинг:** эффективность промокодов, источники трафика (интеграция с Яндекс Метрикой), реферальная программа — стоимость привлечения
5. **Экспорт** всех отчётов в Excel/CSV, настраиваемые дашборды для руководителя

---

### 3.13 Сотрудники и роли (`/admin/team`)

- Список сотрудников: имя, роль, email, статус (активен/заблокирован), дата последнего входа
- **Роли и права доступа** (RBAC): гибкая настройка доступа к разделам панели по ролям (Владелец / Менеджер заказов / Контент-менеджер / Менеджер колеровки / Оператор опта / Бухгалтер / Модератор)
- Журнал действий сотрудников (аудит-лог): кто, что, когда изменил — обязательно для операций с заказами, ценами, бонусами клиентов

---

### 3.14 Настройки платформы (`/admin/settings`)

- **Общие:** название магазина, реквизиты юрлица, контакты, соцсети (для футера витрины)
- **Доставка:** настройка интеграций СДЭК/Boxberry/Почта России API, тарифы, зоны доставки
- **Оплата:** параметры наложенного платежа (актуально уточнить схему по 54-ФЗ с бухгалтером)
- **Уведомления:** шаблоны SMS/email/push для клиентов (подтверждение заказа, смена статуса, начисление бонусов) и для сотрудников
- **Интеграции:** Яндекс Метрика, Яндекс Карты, Typesense/Meilisearch (переиндексация каталога), 1С (при подключении)
- **Юридическое:** редактирование текстов политики конфиденциальности, оферты, условий программы лояльности (без разработчика)
- **Резервное копирование и логи системы**

## 4. Технические требования

| Слой | Технология |
|---|---|
| Фронтенд | Next.js 14+ (App Router) + TypeScript |
| UI | Tailwind CSS + shadcn/ui, таблицы — TanStack Table |
| Бэкенд | NestJS или Laravel (единая кодовая база с клиентской частью) |
| БД | PostgreSQL (общая с витриной, разграничение доступа на уровне ролей) |
| Авторизация | RBAC + обязательная 2FA для сотрудников |
| Графики/аналитика | Recharts или ECharts |
| Файлы/документы | Яндекс Object Storage (те же бакеты, что и витрина) |
| Экспорт | Генерация Excel/CSV/PDF (накладные, УПД, прайс-листы) |
| Доставка | СДЭК API, Boxberry API, Почта России API (общие интеграции с витриной) |
| Аудит | Отдельная таблица логов действий (кто/что/когда), неизменяемая |

**Требования РФ:**
- SSL/TLS, доступ к панели только по защищённому каналу, рекомендуется ограничение по IP/VPN для критичных ролей
- Хранение и обработка ПД клиентов — на серверах в РФ (152-ФЗ, ст. 18.1)
- Журнал согласий клиентов на обработку ПД должен быть доступен для просмотра в карточке клиента
- Разграничение доступа: сотрудник видит только те разделы и данные, которые нужны для его роли (принцип минимальных привилегий)
- Аудит-лог хранится не менее срока, установленного внутренней политикой компании, и недоступен для редактирования/удаления пользователями панели

## 5. Приоритет разработки

**MVP:** Авторизация с 2FA → Дашборд (базовые метрики) → Заказы (список + карточка + смена статусов) → Каталог (CRUD товаров + остатки) → Клиенты (просмотр + бонусы вручную)

**Phase 2:** Программа лояльности (настройка уровней/правил) → Оптовые клиенты → Заказы на колеровку → Отзывы/модерация → Роли и права доступа → Аудит-лог

**Phase 3:** Блог/контент-менеджер → Промокоды и акции → Расширенная аналитика (LTV, ABC-анализ, маркетинг) → Интеграция с 1С → Настраиваемые дашборды
